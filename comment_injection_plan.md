# Scratch HUD Coach: 가이드북 기반 주석 주입 및 관리 기능 구현 플랜

본 문서는 **📋 가이드북(Guidebook)** 탭에서 AI 생성 JSON 스키마를 붙여넣어 스프라이트별 개별 주석 주입 및 텍스트 가이드 복사를 수행하는 통합 기능의 상세 설계 및 구현 로직을 기술합니다.

---

## 🎯 1. 핵심 목표 및 UX 개선 철학

1. **엄격한 기능 격리 (Tab Isolation)**:
   * **블록 주입기 탭**: 오직 코드(블록 구조) 자체를 주입하고 청소하는 기능으로 제한. `[🧩 블록 삭제]`를 포함한 워크스페이스 코드 초기화 기능 전담.
   * **가이드북 탭**: 코드 완성 이후의 피드백, 설명 주석(Comment) 생성, 개별 스프라이트 주석 주입 및 문서화 텍스트 복사를 처리하는 교육 도구로 일원화. `[📝 주석 삭제]` 기능만을 단독 제공하여 가이드 청소 도구 전담 (실수로 블록 코드가 지워지는 오작동 원천 차단).
2. **매핑 안정성 극대화 (JSON-based Mapping)**:
   * 자유 형식의 한글 가이드 설명 텍스트 대신 구조화된 **주석 JSON** 형식을 중간 매개체로 채택하여 오작동 없는 100% 블록 매핑을 보장합니다.
3. **사용자 제어권 강화 (Granular Control)**:
   * 프로젝트 전체에 주석을 한 번에 쏟아 넣는 대신, 사용자에게 파싱된 스프라이트 카드를 노출하고 **스프라이트 단위로 주입 및 복사**를 처리할 수 있게 합니다.
4. **로컬 가이드 생성 비중 최소화 (Low Priority for Local Gen)**:
   * 로컬 규칙 기반으로 텍스트 가이드를 뽑아주던 `[🚀 주석 가이드 생성]` 기능은 보조적인 간이 기능으로 비중을 낮춥니다(버튼 크기를 줄이고 차분한 톤으로 변경).
   * 사용자가 복사한 프롬프트를 바탕으로 외부 AI로부터 구조화된 JSON 결과를 받고, 이를 활용해 **실시간 자동 주석 주입 및 정제 텍스트 복사**를 수행하는 흐름에 UX의 메인 포커스를 둡니다.
5. **AI 프롬프트 가이드 접기/펼치기 레이아웃 최적화 (Optimized Folding)**:
   * 대형 프롬프트 내용물(`textarea`) 영역만 기본적으로 접어두며, `[ ] 프로젝트 내 주석 포함` 체크박스와 `[📋 가이드 전체 복사]` 버튼은 **접기 여부와 상관없이 항상 화면에 노출**되게 구성하여 클릭 불필요성을 없애고 접근 속도를 높입니다.

## 📄 2. AI 출력용 JSON 스키마 스펙

가이드북 탭에서 프롬프트를 통해 외부 AI(예: ChatGPT)에게 요청하여 반환받는 JSON 규격은 다음과 같습니다.

```json
{
  "스프라이트 이름 (예: Sprite1)": [
    {
      "opcode": "event_whenflagclicked",
      "comment": "초록색 깃발 버튼을 클릭했을 때 이 스크립트가 실행되기 시작합니다."
    },
    {
      "opcode": "motion_movesteps",
      "comment": "앞으로 10만큼 스프라이트를 전진시킵니다."
    }
  ],
  "Stage": [
    {
      "opcode": "event_whenbroadcastreceived",
      "comment": "'게임 시작' 신호를 받으면 배경 음악을 재생합니다."
    }
  ]
}
```

## 🎨 3. UI/UX 레이아웃 설계

가이드북 탭의 **4단계 영역** 및 관리 도구 배치와 블록 주입기 탭의 도구 배치입니다.

### [📋 가이드북 탭 하단]
```
+---------------------------------------------------------+
|  📄 4단계: AI 주석 가이드북 연동                         |
+---------------------------------------------------------+
| AI 가이드북 주석 JSON을 아래에 붙여넣으세요.             |
| [=====================================================] |
| [ { "Sprite1": [ ... ] }                              ] |
| [=====================================================] |
|                                                         |
| ─── 파싱된 가이드북 목록 (자동 렌더링) ──────────────── |
|                                                         |
| ■ Sprite1 (스프라이트 1)                                |
|   ├─ 시작 설정: 초록색 깃발 클릭 시 시작합니다.          |
|   └─ 10만큼 움직이기: 앞으로 10 전진합니다.             |
|   [🚀 이 스프라이트에 주석 주입]   [📋 텍스트 가이드 복사] |
|                                                         |
| ■ Stage (무대)                                         |
|   └─ 신호 받았을 때: 게임 시작 신호 시 음악 재생         |
|   [🚀 이 스프라이트에 주석 주입]   [📋 텍스트 가이드 복사] |
|                                                         |
+---------------------------------------------------------+
|  🗑️ 가이드 주석 초기화 도구                             |
+---------------------------------------------------------+
| [ 🗑️ 모든 스프라이트의 주석(말풍선)만 일괄 삭제 ]        |
| * 블록 코드는 안전하게 보호되며 주석 잔상만 즉시 청소합니다. |
+---------------------------------------------------------+
```

### [⚡ 블록 주입기 탭 상/하단]
```
🤖 AI 프롬프트 가이드 (실시간 환경 연동)
   [ ▼ 프롬프트 텍스트 보기 (접기/펼치기 토글) ]
   +---------------------------------------------+
   | (접힘/펼침 가능한 텍스트 영역: 기본적으로 숨김) |
   +---------------------------------------------+
   [x] 프로젝트 내 주석(구현 지침) 포함     <-- 항상 노출
   [ 📋 가이드 전체 복사 ]                  <-- 항상 노출

... 중략 (블록 주입 영역) ...

+---------------------------------------------------------+
|  🗑️ 에디터 블록 초기화 도구                             |
+---------------------------------------------------------+
| [ 🗑️ 모든 스프라이트의 코드 블록 전체 삭제 ]            |
| * 현재 워크스페이스 상의 모든 코드를 영구 제거합니다.      |
+---------------------------------------------------------+
```

## 🛠️ 4. 세부 구현 작업 계획 (Roadmap)

### 1단계: AI 프롬프트 수정 및 템플릿 보완
* `content/content.js` 내부의 프롬프트 가이드 생성 로직(`rebuildAiPrompt`)에 가이드북 전용 지침을 업데이트합니다.
* AI가 한글 텍스트뿐만 아니라, 특정 키에 블록 정보와 한글 설명을 맵핑한 JSON 포맷을 반환하도록 유도하는 전용 프롬프트 템플릿을 탑재합니다.

### 2단계: 가이드북 탭 UI 리팩토링 (`hud_template.js`)
* 가이드북 탭 3단계 영역의 `[🚀 주석 가이드 생성]` 버튼 디자인을 흐린 색상의 작은 버튼(Secondary)으로 하향 리디자인합니다.
* 4단계 영역의 `textarea`를 **AI 주석 JSON 입력 창**으로 전환합니다.
* 파싱된 JSON을 기반으로 스프라이트별 리스트, `[🚀 즉시 주입]`, `[📋 텍스트 복사]` 버튼이 동적으로 렌더링될 컨테이너 `<div id="hud-guidebook-card-list">`를 신설합니다.
* 가이드북 최하단에 `[🗑️ 가이드 주석만 지우기]` 단독 청소 버튼을 배치합니다.

### 3단계: JSON 수신 및 스프라이트 카드 생성기 구현 (`content.js`)
* 가이드북 입력창의 `input`/`change` 이벤트를 감지하여 실시간 JSON 문법 및 스키마 검사를 수행합니다.
* 유효한 스키마가 감지되면, 해당 JSON에 명시된 스프라이트들을 순회하며 UI 카드를 동적으로 렌더링하는 템플릿 헬퍼를 추가합니다.

### 4단계: 실시간 에디터 주석 주입 로직 연동 (`page_bridge.js`)
* 특정 스프라이트의 `[🚀 즉시 주입]` 클릭 시, 해당 스프라이트의 이름과 코멘트 목록(opcode 및 comment 맵)을 `postMessage`로 전달합니다.
* `page_bridge.js` 내에서 대상 스프라이트(Active 여부 확인)의 Blockly Workspace를 가져와 기존 블록의 `opcode`를 매핑하고, 매칭되는 블록들에 `Blockly.Comment` 풍선을 직접 실시간 주입합니다.
  * **주입 조건**: 기존 에디터 블록 순서대로 1:1 매칭하여 순서대로 주석을 할당합니다.
  * **주석 연출**: 뷰 가독성을 위해 기본적으로 말풍선은 접힌 상태(`minimized: true`)로 주입하되, 필요 시 클릭하여 활성화하도록 설정합니다.

### 5단계: 텍스트 포맷터 구현 및 복사 기능 바인딩 (`ui_utils.js`)
* `[📋 텍스트 복사]` 클릭 시, JSON 객체를 한글 단계별 순서(예: `1. 시작 설정: ... \n 2. 동작 설정: ...`)로 깔끔하게 파싱해 텍스트 문서로 변환하고 클립보드 복사를 트리거합니다.

### 6단계: 워크스페이스 관리 도구 분리 및 개편
* 기존의 통합형 `🗑️ 워크스페이스 관리 도구` (블록/주석 체크박스 선택 방식)를 해체합니다.
* **가이드북 탭**: `[🗑️ 가이드 주석만 지우기]` 단독 버튼을 사용하여, 블록 코드는 안전하게 유지한 채 모든 스프라이트의 `comments` 데이터만 깨끗하게 초기화합니다.
* **블록 주입기 탭**: `[🗑️ 워크스페이스 블록 지우기]` 단독 버튼으로 개편하여, 모든 스프라이트의 `blocks` 데이터만 일괄 제거하도록 설정합니다. (단독 주석 삭제 체크박스를 제거하여 UI 복잡도 최소화)

### 7단계: 블록 주입기 프롬프트 가이드 접기 레이아웃 개편
* 블록 주입기 상단의 `AI 프롬프트 가이드` 아코디언 컴포넌트를 수정하여, 접히고 펼쳐지는 대상을 프롬프트 `textarea` 영역으로만 축소합니다.
* `프로젝트 내 주석 포함` 체크박스와 `📋 가이드 전체 복사` 버튼은 아코디언 외부 영역에 상시 노출하여 프롬프트가 접힌 상태에서도 즉각 복사 및 조건 제어가 가능하도록 UI를 개편합니다.

## 🔒 5. 안전 정책 및 검증 계획 (Validation)

1. **기존 블록 파괴 금지**:
   * 이 기능은 블록의 삭제/추가/변경 없이 오직 **주석 내용물만 에디터에 주입**하므로 기존 제작된 학습용 스크립트에 절대 악영향을 주지 않아야 합니다.
2. **스프라이트 불일치 예방**:
   * 주입하려는 스프라이트가 현재 프로젝트에 없는 경우 즉시 경고창을 띄우고 작업을 안전하게 중단시킵니다.
3. **매핑 실패 대비**:
   * 에디터 내 블록 개수/종류와 JSON 내 주석 정의 개수/종류가 맞지 않을 경우, 가능한 블록까지만 매핑 주입하고 초과분은 무시하는 안전망을 구축합니다.
